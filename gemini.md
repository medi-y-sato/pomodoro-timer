# Gemini開発ガイドライン (pomodoro-timer)

このドキュメントは、AIアシスタントGeminiが `pomodoro-timer` プロジェクトで開発を行う際のルールとガイドラインを定めたものです。

## 1. 基本的な方針

- **目的:** このプロジェクトは、シンプルで直感的に使えるポモドーロタイマーを提供することを目的とします。
- **シンプルさの維持:** 機能を追加する際は、このシンプルさが損なわれないかを常に考慮してください。UIが複雑になりすぎないように注意してください。

## 2. 技術スタックと構成

- **言語:** TypeScript
- **ディレクトリ構成:**
    - `src/`: TypeScriptのソースコードを格納する場所です。開発はこのディレクトリのファイルに対して行います。
    - `docs/`: GitHub Pagesで公開されるファイルを格納する場所です。このディレクトリ内のファイルを直接編集しないでください（`dist/`内の生成物を除く）。
    - `docs/dist/`: `src/`のTypeScriptをコンパイルした結果のJavaScriptファイルが格納されます。

## 3. 開発ワークフロー

1. **初回セットアップ:** `npm install` を実行して依存関係をインストールします。
2. **コーディング:** `src/script.ts` を編集します。
3. **ビルド:** 変更後、必ず `npm run build` を実行して、`docs/dist/script.js` を更新します。
4. **動作確認:** `docs/index.html` をブラウザで開いて確認します。

## 4. コーディング規約

- **TypeScript:** `tsconfig.json` の `"strict": true` を維持し、厳格な型チェックを遵守してください。
- **DOM要素の取得:** `document.getElementById` などでDOM要素を取得する際は、必ず適切な型アサーション（例: `as HTMLInputElement`）を行ってください。
- **可読性:** 状態管理（`state`オブジェクト）や関数の責務分割など、既存のコードの設計思想を踏襲し、可読性の高いコードを維持してください。

## 5. Git & GitHubフロー

このリポジトリでは **GitHub Flow** を採用します。

1.  **`main`ブランチ:** `main`ブランチは常にデプロイ可能な状態を維持します。直接のプッシュは禁止です。
2.  **ブランチの作成:** 機能追加や修正を行う際は、必ず`main`ブランチから最新の状態で、分かりやすい名前のフィーチャーブランチを作成してください。（例: `feat/add-new-feature`, `fix/fix-a-bug`）
3.  **Pull Request:** 作業が完了したら、`main`ブランチへのPull Requestを作成してください。変更内容を簡潔に説明し、レビューを依頼する形を取ります。（今回はセルフレビューで構いません）
4.  **マージ:** Pull Requestが承認されたら、`main`ブランチにマージします。
5.  **コミットメッセージ:** [Conventional Commits](https://www.conventionalcommits.org/) の規約に従ってください。
    - `feat:`: 新機能の追加
    - `fix:`: バグ修正
    - `docs:`: ドキュメントの変更
    - `style:`: コードの動作に影響しないスタイルの変更
    - `refactor:`: リファクタリング
    - `test:`: テストの追加・修正
    - `chore:`: ビルドプロセスや補助ツールの変更

## 6. デプロイ

- `main`ブランチにマージされると、GitHub Actions経由（またはGitHub Pagesの機能により）、`docs`ディレクトリの内容が自動的に公開サイトにデプロイされます。
